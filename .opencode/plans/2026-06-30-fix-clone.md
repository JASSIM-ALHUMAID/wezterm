# Fix clone and spawn workspace

**Goal:** Fix workspace clone/spawn so the new workspace gets a numbered name based on the source (e.g., `wezterm-2`) and the user lands in it immediately.

**Problems:**
1. `next_numbered_workspace_name("workspace")` uses literal "workspace" prefix — should use source name
2. `SwitchToWorkspace` on the original window reference doesn't reliably switch to the new workspace — need to use the gui_window from the spawned mux window

**Files to modify:**
- `wezterm/workspaces/prompts.lua`

---

## Task 1: Fix `spawn_associated_workspace`

**Change prefix from "workspace" to source workspace name.**

In `spawn_associated_workspace` (line ~130), change:
```lua
local target_name = M.next_numbered_workspace_name("workspace")
```
to:
```lua
local current_workspace = window:active_workspace()
local target_name = M.next_numbered_workspace_name(current_workspace)
```

Also fix the switch — after `SpawnCommandInNewTab`, get the gui window from the new mux window and focus it:
```lua
-- Replace the SwitchToWorkspace call with:
local _, _, new_mux_win = wezterm.mux.spawn_window({
    workspace = target_name,
    cwd = cwd,
    args = args,
})
local gui_win = new_mux_win:gui_window()
if gui_win then
    gui_win:focus()
end
```

Actually, this would create two windows. Instead, keep `SpawnCommandInNewTab` and switch via the gui window approach. Simpler: just use `wezterm.mux.set_active_workspace(target_name)` after spawning.

Final approach for both functions:
1. Spawn the tab/window in the target workspace
2. Call `wezterm.mux.set_active_workspace(target_name)` to switch

---

## Task 2: Fix `clone_current_workspace`

**Same prefix fix + reliable switch.**

Change:
```lua
local target_name = M.next_numbered_workspace_name("workspace")
```
to:
```lua
local target_name = M.next_numbered_workspace_name(source_name)
```

After creating all tabs/panes, switch with:
```lua
wezterm.mux.set_active_workspace(target_name)
```

Remove the `window:perform_action(act.SwitchToWorkspace(...))` call — use `set_active_workspace` instead.
